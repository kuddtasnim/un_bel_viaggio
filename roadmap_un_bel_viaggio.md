## Что есть сейчас (честный аудит)

Твой сайт — это один файл `index.html`. Вот что важно знать:

- **Данные не сохраняются.** Все путешествия (`trips`) прописаны прямо в коде как массив. При перезагрузке страницы всё сбрасывается.
- **AI — имитация.** Кнопка «Generate plan» запускает `setTimeout` на 600ms и фильтрует локальную базу данных по заранее введённым местам. Никакого реального AI нет.
- **Карта — декорация.** Секция Maps показывает CSS-сетку с иконками. Это не настоящая карта.
- **Регистрации нет.** Имя «Tasnim» вписано в HTML вручную.
- **Рефлексия не сохраняется.** После заполнения вопросов данные исчезают.

## Шаг 1 — Регистрация и вход (Auth)

**Что изменяем в HTML:**  
Добавляем модальное окно поверх существующего дизайна — форма входа/регистрации. При успешном входе имя пользователя подставляется динамически вместо "Tasnim".

**Что строим на сервере:**

```
POST /api/auth/register   — создать аккаунт (email + пароль)
POST /api/auth/login      — войти, получить JWT токен
POST /api/auth/logout     — выйти
GET  /api/auth/me         — проверить, кто вошёл
```

**Как это работает:**
1. Пользователь вводит email + пароль
2. Сервер хеширует пароль через `bcrypt` — в базу никогда не попадает сам пароль
3. Выдаётся JWT токен (живёт 7 дней)
4. Токен хранится в `httpOnly cookie` — защита от XSS атак
5. Каждый последующий запрос к API проверяет этот токен

## Шаг 3 — API для данных путешествий

```
GET    /api/trips           — загрузить все путешествия пользователя
POST   /api/trips           — создать новое путешествие
PUT    /api/trips/:id       — обновить путешествие
DELETE /api/trips/:id       — удалить путешествие

GET    /api/trips/:id/logistics     — загрузить логистику
POST   /api/trips/:id/logistics     — добавить пункт
PUT    /api/trips/:id/logistics/:lid — обновить пункт

POST   /api/reflections     — сохранить рефлексию
GET    /api/reflections      — история рефлексий пользователя

POST   /api/notes           — сохранить заметку
GET    /api/trips/:id/notes — заметки по конкретному путешествию

GET    /api/reminders       — загрузить напоминания
POST   /api/reminders       — создать напоминание
PUT    /api/reminders/:id   — отметить как выполнено / обновить
DELETE /api/reminders/:id   — удалить
```

**Что изменится во фронтенде:**  
Вместо массива `trips = [...]` в начале JS будет `fetch('/api/trips')` при загрузке страницы. Все кнопки «Save» отправляют данные на сервер. Это примерно 150-200 строк изменений в JS — я напишу это целиком.

## Шаг 4 — AI модуль (Plan tab)

**Текущее состояние:** `runPlan()` — это имитация с заранее введёнными данными только по Paris, Lisbon, Tokyo, Prague, Beijing, Istanbul, Rome.

**Что строим:** реальный AI через [Hugging Face Inference API](https://huggingface.co/inference-api) — бесплатный токен, без кредитной карты.

**Модель:** `mistralai/Mistral-7B-Instruct-v0.2` или `google/flan-t5-xxl`  
(лучше Mistral — он понимает запросы на русском и итальянском)

**Как будет работать:**

1. Пользователь выбирает город, интересы, темп
2. Браузер отправляет: `POST /api/plan` с этими параметрами
3. Сервер формирует промпт:  
   *"Create a travel itinerary for [city]. Interests: [coffee, books, quiet spots]. Pace: slow. Return JSON with places: name, address, category, walking_time, price_range, short_note."*
4. Сервер отправляет промпт в Hugging Face
5. Разбирает ответ и возвращает браузеру готовый JSON
6. Браузер отображает план так же, как сейчас — без изменений UI

**Регистрация на Hugging Face:** бесплатно на huggingface.co → Settings → Access Tokens → New Token.

## Шаг 5 — Google Maps

**Текущее состояние:** CSS-сетка с иконками. Не кликабельно, не интерактивно.

**Что строим:** настоящая Google Maps с поиском и маркерами.

**Стоимость:** Google даёт $200 кредитов в месяц бесплатно. При обычном использовании одним человеком это фактически бесплатно.

**Регистрация:**
1. console.cloud.google.com → создать проект
2. Включить Maps JavaScript API и Places API
3. Создать API ключ
4. Ограничить ключ своим доменом (безопасность)

**Что добавляем в HTML:**  
Заменяем `.map-box` с CSS-сеткой на реальный `<div id="map">` + подключаем Google Maps JS SDK. Маркеры автоматически берутся из путешествий пользователя.

## Шаг 6 — Защита данных (GDPR)

Поскольку ты собираешь личные данные (рефлексии, заметки, эмоции), нужно соблюдать базовые требования:

**Что добавим на сайт:**

1. **При регистрации:** чекбокс «Я принимаю политику конфиденциальности» — обязательный. Дата согласия записывается в базу.

2. **Страница «Мои данные»** (добавим вкладку в профиль):
   - «Скачать мои данные» — экспорт всего в JSON файл
   - «Удалить аккаунт» — полное удаление всех данных из базы

3. **Страница политики конфиденциальности** — отдельная страница `/privacy`. Я её напишу. Ключевые пункты:
   - Какие данные собираются (email, путешествия, рефлексии)
   - Как хранятся (зашифрованные пароли, защищённые запросы)
   - Третьи стороны: Hugging Face (AI запросы без личных данных), Google Maps
   - Право на удаление

4. **Технические меры:**
   - Пароли: только bcrypt-хеш, никогда plaintext
   - Токены: httpOnly cookies, не localStorage
   - HTTPS: Railway даёт автоматически
   - Данные одного пользователя недоступны другому (проверка user_id на каждом API запросе)

## Шаг 7 — Деплой на Railway

**Railway** — хостинг, который читает твой GitHub и автоматически публикует сайт при каждом обновлении кода.

**Шаги:**
1. Зарегистрироваться на [railway.app](https://railway.app) (бесплатный тариф есть)
2. Нажать «New Project» → «Deploy from GitHub repo»
3. Выбрать `un_bel_viaggio`
4. Добавить переменные окружения (Settings → Variables):
   ```
   JWT_SECRET=<случайная строка 32+ символов>
   HF_TOKEN=<твой токен Hugging Face>
   GOOGLE_MAPS_KEY=<твой ключ Google>
   NODE_ENV=production
   ```
5. Railway сам запустит `npm start` и даст тебе URL вида `un-bel-viaggio.up.railway.app`
6. Можно привязать свой домен (если есть)

**Стоимость Railway:** бесплатный тариф даёт $5 кредитов в месяц, этого хватает для небольшого сайта. Платный план — $5/месяц с большим лимитом.
